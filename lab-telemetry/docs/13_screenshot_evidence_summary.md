# Screenshot and Evidence Summary

The screenshots in `Coding Fest.zip` are retained as visual evidence of the lab build process. Codex should not need to parse image pixels for normal automation work; use the Markdown memory and command history as the authoritative text context.

## What the screenshots and command history collectively support

- Multipass DB/Auth/Web infrastructure was created and networked.
- PostgreSQL was configured on `db-server`.
- Auth Server and Web Server Python environments were created.
- Auth Server FastAPI `/health` and `/login` were tested.
- Web Server FastAPI routes `/`, `/search`, `/login`, `/admin`, and `/health` were tested.
- JSON logs were observed in `auth.log` and `webapp.log`.
- `auth-lab.service`, `web-lab.service`, `nginx`, and `postgresql` were verified as active in later health checks.
- Linked Web/Auth evidence was verified through matching web login attempts and auth login results.
