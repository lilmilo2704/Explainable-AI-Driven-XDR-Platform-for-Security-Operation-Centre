# Documentation Patterns

- Current implementation and executable configuration outrank historical plans.
- Validated model metadata and dataset artifacts outrank stale README claims.
- `docs/00_PROJECT_CURRENT_STATUS.md` and `docs/PROJECT_STATUS_EVIDENCE.md` are the current status anchors.
- Mark historical four-class scope as reference unless implementation and trained models support it.
- Keep limitations visible: controlled lab data, mainly application-layer single-source service stress, no production-grade DDoS claim.
- Documentation-only changes must not alter runtime code or protected artifacts.
- When changing contracts, update status, contract docs, workflows, and tests together.
